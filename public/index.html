API Project: Timestamp microservice

The api gets passed a string as a parameter, and checks to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016)

If it does, it returns both the Unix timestamp and the natural language form of that date.

If it does not contain a date or Unix timestamp, it returns null for those properties

Usage

Requesting [project url]/api/timeservice/2018-04-29 or
[project url]/api/timeservice/152496400 returns 


  {
    unixtime: 1524956400,
    natural: "April 29th 2018"
  }
